# EWT v2 runtime and parity fixes - 2026-05-24

This patch addresses runtime/UX issues reported during Windows testing.

## Fixed areas

- Added Windows elevation manifest so the release executable requests administrator privileges through UAC.
- Converted long-running backend commands to async `spawn_blocking` wrappers so the Tauri UI is not blocked by command execution.
- Added global operation overlay with a visible status while commands are running.
- Separated startup dashboard loading from slower disk-health refresh.
- Disk Health is now loaded on-demand when the page is opened or when Refresh is pressed.
- Added scroll-to-top behavior when switching pages.
- Added a scrollable sidebar so all menu entries remain accessible at the default window size.
- Removed the standalone Restart Explorer action from Tweaks.
- Classic context menu toggle now uses direct registry commands and restarts Explorer as a detached process.
- Fixed registry syntax for Start with Windows, Mouse Acceleration and NumLock actions.
- System Restore cleanup now uses `vssadmin delete shadows /all /quiet` and is marked as a high-impact action.
- Battery report now asks for an output path and defaults to the real Windows Documents folder instead of assuming `C:\Users\User\Documents`.
- Install page now groups apps by category, scans installed apps, displays installed badges, shows install operation status, and clears selected apps after a successful bulk operation.
- Expanded app catalog with legacy v1.3.5 catalog entries.
- Preserved Tauri icon bundle configuration and icons folder.

## Notes

- The frontend TypeScript/Vite build was validated with `npm run build`.
- Rust/Tauri bundling still needs to be validated on Windows because this environment does not include Cargo/Rust.
- Windows features such as Recall, Copilot policies, BitLocker, HAGS, VBS, restore-point deletion and Windows app removal may vary by Windows edition and build.
